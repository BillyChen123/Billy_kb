# 使用说明

这里是 Billy_kb 的操作手册区。新成员、维护者和 Agent 都应该先从这里理解协作方式。

## 推荐阅读顺序

人类成员：

1. [如何接入Obsidian](./%E5%A6%82%E4%BD%95%E6%8E%A5%E5%85%A5Obsidian.md)
2. [写作与整理规范](./%E5%86%99%E4%BD%9C%E4%B8%8E%E6%95%B4%E7%90%86%E8%A7%84%E8%8C%83.md)
3. 根目录 [README](../README.md)

Agent：

1. 根目录 [README](../README.md)
2. 根目录 [AGENTS](../AGENTS.md)
3. [如何让Agent读取知识库](./%E5%A6%82%E4%BD%95%E8%AE%A9Agent%E8%AF%BB%E5%8F%96%E7%9F%A5%E8%AF%86%E5%BA%93.md)
4. [写作与整理规范](./%E5%86%99%E4%BD%9C%E4%B8%8E%E6%95%B4%E7%90%86%E8%A7%84%E8%8C%83.md)

## 文件说明

- `如何接入Obsidian.md`：从安装 Git 到配置 Obsidian Git 的完整接入流程。
- `如何让Agent读取知识库.md`：Agent 检索、引用和整理知识库时的具体步骤。
- `写作与整理规范.md`：frontmatter schema、命名规则、质量标准、投稿流转规则。

## 维护规则

- 接入流程变化时，优先更新 `如何接入Obsidian.md`。
- 目录结构变化时，同时更新根目录 `README.md` 和本目录说明。
- Agent 读写规则变化时，同时更新 `AGENTS.md` 和 `如何让Agent读取知识库.md`。
- 写作格式和质量门槛变化时，更新 `写作与整理规范.md`。
