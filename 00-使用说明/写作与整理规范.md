# 写作与整理规范

Billy_kb 不要求人手填复杂模板。人负责推进，Agent 负责整理、同步 memory、沉淀 lesson。

## 最小 frontmatter

```yaml
---
title: 文档标题
type: note
status: draft
domain: project
project: 项目名
maintainer: Billy
updated: 2026-06-07
tags:
  - example
agent_reading: cautious
---
```

常用 `type`：

- `memory`
- `project`
- `note`
- `lesson`
- `paper`
- `tutorial`
- `how-to`
- `reference`
- `explanation`
- `skill`
- `mcp`
- `pattern`
- `discussion`

常用 `status`：

- `inbox`
- `draft`
- `active`
- `reviewed`
- `deprecated`

## memory.md 写法

`memory.md` 用轻提示，不用重表格：

```markdown
# memory

Agent 必读。每次推进项目后，如果当前状态、关键判断、下一步发生变化，更新这里。

- 当前目标：
- 当前状态：
- 关键判断：
- 不要重复踩的坑：
- 下一步：
- 关键链接：
```

## 模板规则

`30-模板/` 的模板主要给 Agent 使用。人不需要手动填满所有字段。

如果信息不足，Agent 应写“待核验”，不要编造。

## 质量检查

Agent 生成或整理文档时，至少检查：

- 是否有 frontmatter？
- 是否能看出这篇文档是什么？
- 是否保留来源或关键链接？
- 是否区分了结论和待核验？
- 项目推进后是否更新了 `memory.md`？
